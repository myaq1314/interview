# 简单工厂模式

简单工厂模式 不在 GoF 23 种设计模式之列

简单工厂模式，有一个具体的工厂类，可以生成多个不同的产品，属于创建型设计模式

# 定义

定义一个创建产品对象的工厂接口，将产品对象的实际创建工作推迟到具体子工厂类当中。

# 角色

1、具体工厂类

    生成多个不同的产品

2、产品接口

    产品规范

3、具体产品

    工厂类实际生成的类对象

# 优点

1、一个调用者想创建一个对象，只要知道其名称就可以了

2、扩展性高，增加一个产品，只要扩展一个工厂类就可以了

3、屏蔽产品的具体实现，调用者只关心产品的接口。

# 缺点

1、每次增加一个产品，都要增加一个具体类和对象实现工厂，使得系统中类的个数成倍增加，

在一定程度上增加了系统的复杂度，增加了系统具体类的依赖。

# 注意事项

复杂对象适合使用简单工厂模式，而简单对象，特别是只需要通过 new 就可以完成创建的对象，无需使用简单工厂模式。

# 应用实例

您需要一辆汽车，可以直接从工厂里面提货，而不用去管这辆汽车是怎么做出来的，以及这个汽车里面的具体实现。
