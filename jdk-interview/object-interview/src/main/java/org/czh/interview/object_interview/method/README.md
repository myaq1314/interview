# 方法调用
    
    使用 javap -v Test.class 查看Class文件的指令集

    class字节码文件中的方法的调用，都只是符号引用，不是直接引用（方法在实际运行时，内存布局中的入口地址）

### 虚方法与非虚方法

    非虚方法：在类加载的时候把符号引用解析为直接引用
    虚方法：在运行期间才动态绑定方法

### 解析

    在类加载的解析阶段，会有一部分的符号引用转化为直接引用
    该解析的前提是：方法在程序真正运行之前已经有一个可确定的调用版本，
    并且这个方法的调用版本在运行期是不可改变的
    这类的方法调用称为解析

### 分派

    在运行期间根据方法的实际类型绑定相关的方法，
    这类的方法调用称为分派

#### 静态分派（重载）

    静态类型的变化仅仅在使用时才发生，变量本身的静态类型是不会被改变，并且最终静态类型在编译器是可知的。 实际类型的变化是在运行期才知道，编译器在编译程序时并不知道一个对象的具体类型是什么

    所有依赖静态类型来定位方法执行版本的分派动作称为静态分派。
    静态分派的典型应用就是方法重载。
    静态分派发生在编译阶段
    静态分派的动作由编译器来完成。

#### 动态分派

    在运行期根据实际类型确定方法执行版本的分派过程称为动态分派
    动态分派的典型应用就是方法重写

#### 单分派 和 多分派

方法的接受者与方法的参数统称为方法的宗量

    根据分派基于多少种宗量，将分派分为单分派和多分派
    单分派是根据一个宗量，对目标方法进行选择
    多分派时根据多于一个宗量，对目标方法进行选择。
    
    静态分派属于多分派类型
    动态分派属于单分派类型

### 虚方法表

    JVM采用在类的方法区建立了一个虚方法表（Vritual Method Table，简称 vtable）
        使用需方发表索引来代替元数据查找以提高性能
        invokeinterface指令执行时，使用接口方法表（Inteface Method Table，简称 itable）
    如果子类没有重写父类的某个方法，那么子类的虚方法表里面的地址入口和父类相同方法的地址入口是一致地，都指向父类的实现入口。
    如果子类重写了这个方法，子类方法表中的地址将会替换为指向子类实现版本的入口地址。
    具有相同签名的方法，在父类、子类的虚方法表中都应当具有一样的索引序号，
        当类型变换时，仅需要变更查找的方法表，就可以从不同的虚方法表中按索引转换出所需的入口地址
    虚方法表一般在类加载的连接阶段进行初始化，准备了类的变量初始化值后，虚拟机会把该类的方发表也初始化完毕。

### 绑定机制

    解析调用一定是静态的过程，在编译期间就能完全确定，在类装载的解析阶段就会把涉及的符号引用全部转变为可确定的直接引用，不会延迟到运行期再去完成。
    分派调用则可能是静态分派和动态分派，
    静态链接：解析和静态分派，在编译期间就确定了被调用的方法，且在运行期间不变的调用
    动态链接：在运行期才确定下来的调用
    早期绑定：静态链接过程中的转换
    晚期绑定：动态链接过程中的转换
