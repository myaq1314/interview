CLH锁 基于逻辑队列、非线程饥饿的一种自旋公平锁

AQS 是变种的 CLH锁

## 自旋锁 busy-waiting

一种互斥锁，没有抢到锁的线程会一直自旋等待锁的释放，处于 busy-waiting 的状态

此时等待锁的线程不会进入休眠状态，而是一直忙等待

自旋锁适用于锁占用时间短的场合。

## 互斥锁 sleep-waiting

多个线程并发竞争锁的时候，没有抢到锁的线程会进入休眠状态，即 sleep-waiting

当锁被释放的时候，处于休眠状态的一个线程会再次获取到锁

缺点是这一系列过程需要线程切换，需要执行很多 CPU 指令，同样需要时间

如果 CPU 执行线程切换的事件，比锁占用的时间还长，那么还不如使用自旋锁。

互斥锁适用于锁占用时间长的场合。

### 原理

1、节点内使用 volatile 修饰 的 locked 状态位

    默认false，线程没有获取锁
    true，线程获得锁或线程等待锁

2、初始化

    原子级尾节点 填充空节点，状态位为false
    当前节点线程本地变量，填充空节点
    前置节点线程本地变量，填充空

3、获取锁，设置当前节点状态位true，前继节点为空节点，状态位false， 获取锁

4、释放锁，设置当前节点状态位false，当前节点重新填充空节点

5、等待锁，在前继节点locked状态位 上自旋等待，为false时获取锁
