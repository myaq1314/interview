# 抽象工厂模式

抽象工厂模式 属于 创建型模式

在抽象工厂模式中，接口是负责创建一个相关对象的工厂，不需要显示指定它们的类。每个生成的工厂都能按照工厂模式提供对象

# 优点

1、当一个产品族中的多个对象被设计成一起工作时，它能保证客户端始终只使用同一个产品族中的对象。

# 缺点

1、产品族扩展非常困难，要增加一个系列的某产品，既要在抽象的 Creator 里加代码，又要在具体的里面加代码。

# 注意事项

产品族难扩展，产品等级易扩展。

# 应用实例

您需要一辆汽车，可以直接从工厂里面提货，而不用去管这辆汽车是怎么做出来的，以及这个汽车里面的具体实现。 您需要一些食物，可以直接从商场里面购买，而不用去弄清这些食物是怎么做出来的，以及这些食物的具体实现。
