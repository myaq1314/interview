# 构造器

    默认构造器
        如果没有定义构造器，那么会默认一个无参构造器
    自定义构造器
        如果自定义了构造器，那么会覆盖默认的构造器
    无参构造器
        没有参数的构造器，可以直接通过 new ClassName() 创建一个实例
    有参构造器
        有1个或多个参数的构造器，需要在创建一个实例时，提供参数的值，并传入有参构造器 new ClassName(Integer, Double)
    复制构造器
        只有1个参数，而且这个参数是当前类的一个实例，该构造器会创建一个该实例的副本（浅拷贝）
    私有构造器
        不希望定义的对象被外部创建，将构造器的修饰符改为 private 即可（单例）
    构造器的继承
        子类会默认调用父类无参构造器，
        如果父类没有无参构造器，那么必须在子类构造器的第一行通过 super 关键字指定调用父类的哪个构造器。

## 5.1 构造器的执行熟悉怒

[详见测试代码](ConstructorLearn.java)

    父类静态代码块 (只执行一次，在类加载的时候，如果之前已经执行过了，那么本次不执行)
    子类静态代码块（只执行一次，在类加载的时候，如果之前已经执行过了，那么本次不执行）
    
    父类构造代码块（每次新建实例或子类新建实例，都会执行一次）
    父类构造器（每次新建实例或子类新建实例，都会执行一次）
    子类构造代码块（每次新建实例，都会执行一次）
    子类构造器（每次新建实例，都会执行一次）

    静态方法（类调用，每次调用，都会执行一次）
    构造器才能执行方法（构造器调用，每次调用，都会执行一次）
